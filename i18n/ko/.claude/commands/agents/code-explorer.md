# P2: Code Explorer

**Phase**: 2 - Codebase Exploration
**도구**: Read, Glob, Grep (Write 불가)
**병렬**: 2-3개 동시 스폰
**Output**: 없음 (결과를 Harness에 텍스트로 반환)

---

## 역할

기존 코드베이스를 탐색하여 패턴, 컨벤션, 아키텍처 결정을 파악합니다.
코드를 수정하지 않습니다.

## 병렬 에이전트 프롬프트

### 에이전트 1: 유사 기능 탐색
```
"[요청 기능]과 유사한 기존 기능을 찾아 구현 패턴 분석.
파일 구조, 네이밍 컨벤션, 데이터 흐름 패턴을 식별."
```

### 에이전트 2: 아키텍처 맵핑
```
"[관련 영역]의 아키텍처와 추상화 맵핑.
레이어 구조, 의존성 방향, 진입점을 식별."
```

### 에이전트 3: 패턴 분석 (선택)
```
"UI 패턴, 테스트 접근법, 확장 포인트 식별.
기존 테스트 프레임워크 설정, 컴포넌트 패턴을 파악."
```

## 결과 병합 (Harness에서 수행)

```
1. 각 에이전트가 반환한 핵심 파일 목록 수집
2. 중복 제거
3. 패턴/컨벤션/아키텍처 결정 요약
```

## 반환값

Harness에 반환: 핵심 파일 목록, 패턴 요약, 기술 스택 정보
